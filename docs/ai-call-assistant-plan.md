# AI架電支援・録音分析の設計メモ

## できること

- 施設ごとの情報、状態、温度感、備考をもとに、AIが架電スクリプト案を作る
- 作成したスクリプトを音声読み上げし、架電前にスタッフが確認する
- 通話後の録音を文字起こしし、AIが改善点、次回確認事項、温度感、NG理由を整理する
- 分析結果を備考、架電履歴、次回アクションへ反映する

## 静的版でできる範囲

- 施設ごとのスクリプト欄を追加
- ブラウザの音声読み上げ機能でスクリプトを再生
- 録音ファイルを手動アップロードしてメモ欄に整理

静的HTMLだけでは、AI生成、録音の安全な保存、複数ユーザー権限管理、長時間処理の自動実行には向きません。

## 本番版での推奨構成

- Supabase Storage: 録音ファイルを保存
- Supabase Postgres: スクリプト、文字起こし、AIアドバイス、承認状態を保存
- Supabase Edge Functions: 文字起こしとAI分析をバックグラウンド実行
- フロントエンド: スタッフは録音アップロードとAI提案の確認、管理者は集計を確認

## 注意点

- 通話録音は相手先への通知や同意が必要になる場合があります。
- 録音ファイルには個人情報が含まれるため、管理者だけが閲覧できる権限設計が必要です。
- AIの提案は自動反映ではなく、スタッフ確認後に備考や次回アクションへ反映する設計が安全です。
