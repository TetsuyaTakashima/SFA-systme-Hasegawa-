# 文化施設 営業管理

公立の文化施設を営業先として管理する、ブラウザだけで動く静的Webアプリです。

## できること

- 施設名、管理運営機関、電話番号、アドレス、所在地、担当部署、担当者、ホール情報、営業状況を一覧・詳細で管理
- `list.html` で営業先一覧だけを表示
- 管理画面では営業先一覧テーブルを表示せず、一覧確認と施設追加は営業先一覧ページに集約
- 未確認項目を施設ごとに自動表示
- 状態、担当スタッフ、次回アクションを含めた検索
- 営業先一覧から自主事業、管理運営機関、電話、アドレス、担当、検討時期、次回架電日、施設別の通知日数、次回アクションを直接更新
- 一覧や架電用リストで保存された入力欄を一時的にハイライト表示
- 検討時期を入れると、次回架電日を2か月前に自動設定し、必要に応じて手動変更
- 営業先一覧の列をドラッグ＆ドロップで並び替え、ユーザーごとに保存
- 営業先一覧の `固定列` と `表示列` メニューから、横スクロール時の固定列・表示列・表示プリセットをユーザーごとに設定
- 営業先一覧を `自分の担当のみ`、未割当、担当スタッフ別で絞り込み
- 担当者名を一覧から直接入力
- 架電者を自動記録
- 架電状況を `未架電` または `最終更新日時` で自動表示
- 一覧や架電用リストで入力・変更した時点で架電更新日を自動更新
- 架電通知欄で通知対象を `①` などのバッジで表示
- 通知欄に出す既定日数と、ブラウザのポップアップ通知を出す既定日数を折りたたみ式の詳細設定で管理
- 通知欄の表示を `バッジ`、`バッジ＋日数`、`残り日数`、`架電日` から選択
- 画面右上に、現在ユーザーの通知対象に入っている要確認件数を表示
- 右上の要確認件数から通知対象施設のポップアップ一覧を開き、該当施設へ移動
- 通知設定をユーザーごとに保存し、通知対象を `自分の担当のみ` または `全施設` から選択
- 施設ごとに通知日数を個別設定。空欄の場合は既定日数を使用
- 通知を消す条件を、次回架電日の変更、状態の変更、どちらかの変更から選択
- 温度感 `A〜E`、通知順、非表示順などで営業先一覧を並び替え
- 管理者が温度感 `A〜E` の表示名と色を変更
- 非表示データの絞り込みと表示切り替え
- 管理者が状態名、表示色、架電対象外フラグを管理し、架電対象外の施設をグレー表示
- ホール規模を大・中・小で一覧から直接入力
- 備考をポップアップで入力し、重要メモとして目立たせ、一覧に短いプレビューを表示
- 管理者専用の `history.html` で架電履歴と集計を確認し、重要項目だけに絞り込み
- 管理画面のプルダウンから、ユーザー管理、状態項目管理、温度感設定、CSV取り込みを切り替え
- 管理・スタッフのユーザー切り替えとユーザー管理
- `login.html` でSupabase Authログイン。本番URLでSupabase設定を取得できない場合はログインを停止し、ローカル確認時だけローカルログインを利用
- 管理者ログイン後の管理画面で、名前・ログインID・パスワードによるユーザー作成
- 管理者による状態項目の追加・名称変更・色変更・架電対象外設定・未使用項目の削除
- 次回架電日が近い施設のブラウザ通知
- CSVから営業先を一括取り込み
- 施設名と電話番号または住所が一致する既存データを更新
- Supabase設定済みの場合はSupabaseへ保存し、ローカル確認時はブラウザの `localStorage` に保存
- 登録データをCSVで書き出し

## 使い方

`login.html` をブラウザで開いてログインすると使えます。Supabase運用時は、Supabase Authに作成したユーザーでログインします。初期管理者はログインID `admin`、パスワードはSupabaseで設定したものを使います。

ローカル確認用のログインID `admin`、パスワード `password` は、`file:` 表示または `localhost` のみで使えます。本番環境ではSupabase接続を必須とします。

CSV取り込みは、`sample-facilities.csv` の列名を参考にしてください。列名は「施設名」「管理運営機関」「電話番号」「アドレス」「住所」「担当部署」などの日本語名、または `name`、`phone`、`email`、`address` などの英語名を自動判定します。

全国の私立学校リストは `data/private-schools-national.csv`、小学校・中学校・高等学校などに絞ったリストは `data/private-schools-k12.csv` から取り込めます。どちらも文部科学省の学校コード一覧（令和8年5月1日時点 暫定版）をもとに、CSV取り込み用の列名へ変換しています。元データに電話番号、メール、Webサイト、学校法人名は含まれていないため、それらの項目は空欄です。全件版は約2万件あるため、大量投入する場合はSupabase運用で取り込んでください。

CSV取り込み時は `取り込み種別` で `施設` または `学校` を選べます。`学校` として取り込んだデータは、営業先一覧の `区分` フィルターで学校として絞り込めます。並び順では学校と施設のどちらも優先されません。

小中高等の連絡先補完版は `data/private-schools-k12-contact-enriched.csv` です。東京都、大阪府、神奈川県、千葉県の公開名簿・私学協会サイトから電話番号やWebサイトを補完しています。取得状況は `data/private-schools-k12-contact-summary.md`、再生成スクリプトは `scripts/enrich-school-contacts.py` にあります。

営業先一覧だけを見たい場合は `list.html` を開いてください。

通知はブラウザ通知を使います。現在の静的版ではページを開いている間に通知されます。本番運用では Supabase Edge Functions やメール通知に置き換える想定です。

Supabase設定済みの場合は、入力変更ごとにSupabaseへ保存します。アカウント作成はVercelの `/api/create-user` を通して管理者だけが実行します。

## 想定している営業項目

- 基本情報: 施設名、種別、管理運営機関、所在地、住所
- 連絡先: 電話、FAX、アドレス、Webサイト、担当部署、担当者
- 公演情報: 主ホール名、客席数、大ホール席数、中ホール席数、小ホール席数、得意ジャンル、自主事業（○・△・×）
- 営業状況: 状態、温度感（A〜E）、表示状態、担当スタッフ、最終接触日、架電更新日、架電者、検討時期、次回架電日、通知日数、次回アクション、備考
- 履歴: 架電者、更新日時、更新項目、変更前、変更後

## デプロイ設計

Supabase、GitHub、Vercelで本番化するための設計メモは `docs/deployment-plan.md`、Supabaseの初期スキーマ案は `docs/supabase-schema.sql` にまとめています。既存環境への変更は `supabase/migrations/` のバージョン付きSQLを順に適用してください。`docs/supabase-after-setup.sql` は古い広範なRLSポリシーを含んでいたため、実行用ではなく移行案内に置き換えています。初期管理者の確認には `docs/supabase-admin-check.sql` を使います。

Supabaseの公開接続情報は `cloud-config.js`、ブラウザ側の接続処理は `cloud-client.js` に分けています。Vercelで管理者によるユーザー作成・削除を使う場合は、環境変数 `SUPABASE_URL`、`SUPABASE_ANON_KEY` または `SUPABASE_PUBLISHABLE_KEY`、`SUPABASE_SERVICE_ROLE_KEY` または `SUPABASE_SECRET_KEY` をVercel側にだけ設定してください。
